# v0.1.0 — Reliable two-stage dispatch

The first public release replaces one-shot generation-and-send with a sealed two-stage delivery contract.

## Highlights

- Canonical schema-versioned JSON configuration.
- 06:20 prepare and 07:00 deliver schedules in China Standard Time.
- Atomic ready manifests with file size and SHA-256 verification.
- Exactly-once delivery receipts with Message-ID evidence.
- Failure notifications without stale-content fallback.
- 30-day hard duplicate checks and 7-day source-diversity warnings.
- Five-item art detours across at least three sources.
- Structured JSONL events and Python 3.9/3.12 CI.

## Migration

Existing users should review `dispatch.config.json`, run `scripts/validate_config.py`, recreate the virtual environment with the lock file, and replace the old single schedule with the prepare/deliver pair. Legacy `config.md` field parsing remains read-only for one release.

## Compatibility

`run_daily.py` remains as a compatibility wrapper but now delegates to finalize and deliver. GitHub scheduling remains disabled unless `DISPATCH_ENABLED=true`.

## Known limits

- Editorial quality and web freshness depend on the configured web-search agent.
- GitHub scheduled jobs can queue; the platform cannot guarantee second-level timing.
- Long-term 07:00 SLA requires production history and is not claimed from test results alone.
